A brief about Events and State.

Just adding a temporary button to understand about events and state. Can be acheived by just inputing a button in card.

Pay close attention to the function, we are not writing onClickFunction()

The reason being, whatever is there in { // code } will be executed. Hence just the function name have to be given.

Let’s understand how Component functions are executed.

If we just onClickFunction to title = “New” and then, it wont work. The reason is being that the return function is returning JSX code and also we have to understand how JSX code is being parsed.

Recollect that your custom component is a function and we never call the functions, rather use it as a HTML element. Under the hood, this using is almost like a function call.

We make React aware of this HTML tags and while evaluating, it will call JSX code.

So React understands in this way. It keeps on digs deeper until there are no custom Components. Here, Card is a custom Component. And then ExpenseDate. And so on until there are no custom Components.

The problem is that React only renders once when the application is fire up. So updating is a problem in this.

This is where State comes into picture. A way of telling React that something changed and some certain component needs to be re-evaluated. Hence the concept of State.

State is not a React specific concept, but is an important concept in React. So now to change the title, we are trying to alter the state and informing React that a certain state is being changed and redrawn when it changes.

This function allows us to define values as state and change in this states will cause re-val thou.

It has to be called inside the function.

It is called important Hook (Hooks use the word “use”) → Must be called inside the React Component function. Ex: ExpenseItem.

useState wants default state value, which can be compared to the next state and then triggers the change needed. Therefore, initial value has to be assigned.

This returns us an Array → First value is variable itself, updating Function.

This Hook always returns an Array of two variables → First one is variable value stored and second one is State updating Function.

When we call the State updating Function, not only that the value will be changed, but also JSX code will be revaluated again → So React will render this again.

import ExpenseDate from './ExpenseDate';
import Card from './Card';
import './ExpenseItem.css';
import React, { useState } from 'react';

function ExpenseItem(props) {

	const [title, setTitle] = useState(props.title);
	const onClickFunction = () => {
		setTitle('New Updated title');
		console.log(title);
	}

	return (
		<Card className='expense-item'>
			<ExpenseDate date={props.date} />
			<div className='expense-item__description'>
				<h2>{title}</h2>
				<div className='expense-item__price'>${props.amount}</div>
			</div>
			<button onClick={onClickFunction}>Click me</button>
		</Card>
	);
}

export default ExpenseItem;

Let’s see the useState in detail.

This will register some value as special value → State.

One thing to remember is that useState will create new instances for new instances of Same component type. That is, here, we are having 4 ExpenseItem components, hence a total of 4 instances can be seen, each detached from another.

Hence, seperate States for different instances of Same type of component. Same with the state changing function will change the value of that instance, but not for other.

Why are we using const for useState()?

We are not assigning value using equal to. Instead, we are changing state using State changing function. But the box for useState will be generated only once.
