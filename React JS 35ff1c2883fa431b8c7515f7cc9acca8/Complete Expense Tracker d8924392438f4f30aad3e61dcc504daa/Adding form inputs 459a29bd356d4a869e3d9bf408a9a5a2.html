Once the HTML / CSS for taking User input is done, we can start listening to the User input.

Understand what’s happening here. We are giving an onChange characteristic to input type, which in turn calls enterTextFn which will be called for every stroke / change → Not just submit, but also every change.

So now we have to save the entered value somewhere even though the Component will be revaluated again → Yes, we use useState to achieve this.

There are a couple of ways to solve this. One way is to use useState for all the variables singly and destructuring it into array (3) → and using them. Or else see the current appraoch:

The commented one is the prev approach. Hence, you use setEnteredTitle() to change title and so on. Or else, the current approach is storing all the variables in one object and passing it and it shall also returns an object of data and an object of state changing functions.

The highlighted one is older. But since being asynchronous, it is better to the next approach.

This is better when the current State depends on the previous approach.

So in a brief, there are 3 ways, you can solve the issue of having multiple states in the same component. They are:

  1. Using useState() n times for n variables.
  1. Using useState() singe time by enclosing n variables in single object and while using stateChangingFunction, copy the user input and overwrite.
  1. ( When there is a prev. state dependency ) Passing function with prevState and copying prevState and then overwriting.

For now, we are sticking with 1st step.

Now we should be able to collect data and then do something with it. That can achieved with some HTML only. → Form onSubmit.

The reason for having event.preventDefault is that the default way when submitted by the form, the form reloads. Hence using this prevents browser from refreshing or reloading.
