Complete Briefing.

  • React is a JS Library used for creating UI.
  • React is all about components used to compose UI and update UI.
  • ReactDOM is the interface to the web.
  • React itself doesn’t know the web, only knows how to work with the components and props.
  • ReactDOM is responsible for bringing something onto the screen, works with DOM.
  • React in the end uses virtual DOM, it determines the component tree and what it should look like when updated and handovers it to the ReactDOM.
  • Reevaluating a component is not the same as re-rending the DOM.
  • Components are reevaluated whenever there is a change in state, props or context. And the ReactDOM will only change wherever is needed.
  • Working with the real DOM is an expensive task.
  • This change in the code is called Virutal DOM diffing.
  • ReactDOM only does the change required.
  • If the parent component is executed, so will the children components all will be revaluated and re-executed.
  • We can prevent this unnecesasary re-executing of the components by using React.memo()
  • This is optimsed by wrapping up the export default into React.memo();
  • There is one important point here. Here, when the props are trivial values, like boolean, string, number, etc. React.memo method works for that. But for props having function pointers, this doesn’t work coz the function is created everytime and it is not equal to the previous function.
  • We can prevent this for prop values as objects or function pointers by using the React Hook useCallback that allows us to store a function across function executions → Save a function and do not recreate this again and again.
  • We can use it by wrap the function inside the useCallback as the first argument and the dependencies array as the second argument.
  • Here, it says no dependencies, hence this function is constant. Remember that React.memo has to still be there.
  • The dependencies has to be there when there are some conditions inside the function inside the useCallback().
  • The reason being that the variables declared or used inside the functions are locked by JS and hence it won’t change from what is being declared while rendering the function for the first time. Hence, we give it as the dependency.
  • Components and the state management are the core concepts of React and the connection between state and the components are established by Hooks such as useState, etc done behind the scenes by React.
  • But there is a critical concept here. We are rendering App.js or any other file whenever there is a change in state, props or context. But that means are we re-establishing the connection between them?
  • React makes sure that useState() when executed for the first time, creates a new State variable and hand over it to React which memorizes which component it belongs and initialises with the value we give. For the subsequent calls, React recognises that there is a state for this component and then updates the state, rather than reinitialising the State.
  • Same is true for useReducer also.
  • One critical concept is State scheduling. Whenever there is a state change as above.
  • React will schedule the state change for us. Hence, whenever there is a dependency of one state on the previous state, it is better to use function form of useState.
  • We can also use useMemo() which can does the same work for us.
